The Core Question

「知識」から「知見」を生み出し、

情報収集者から、知的生産者へ

現代は情報洪水（Information Flood）の時代です。単にデータを集めるだけでは、ノイズに埋もれてしまいます。

必要なのは、自然界の「木」のアナロジーを用いた階層的な知識構築フレームワークです。

ROOT / STEM / BRANCH
LAYER 1

ROOT（根）：核心的な問い

すべての始まりは「問い」の設定にあります。ROOTが揺らぐと、知識の木は育ちません。

Core Topic (問い)

知識構築の目的と焦点を決定する。

例：「なぜ日本では財政赤字が続いてもインフレが起きにくいのか？」

Definition

各要素の厳密な定義。
言葉の定義がROOTの一部となる。

ROOTを支える土壌：Facts (Data/Digital)

観察された「事実」がTopic展開を支える。

※事実を一般化したものがSTEM（知識）へと昇華する。

Who When Where What How How much How long How many
LAYER 2

STEM（幹）：構造と論理

主要な視点・ルール

ROOT（問い）を支えるための構造。複雑なテーマを扱いやすいモジュールに分解し、体系化する役割を持ちます。

Backing & Theory

  • ● Backing: STEMを裏付けする論理的関係。
  • ● Theory: Backingの中で、公理的で正しいと論述された関係。
事実からSTEMへの解析・検証
S
Science
科学的検証
T
Technology
技術的実現
E
Engineering
工学的構築
M
Mathematics
数理的証明
LAYER 3

BRANCHING（枝葉）：具体と深掘り

STEM（理論）に具体性と説得力を与えるレイヤー。最も情報量が多く、常に更新され続けます。 ここでは「推論フロー」をルール化します。

1

Experience

自らの経験

2

Experiment

実験・調査結果

3

Exception

例外事例

4

Example

具体例

5

Embodyment

抽象の具体化

これらの枝葉が豊かであるほど、知識の木は「活き活き」とし、強い説得力を持つ。

AI（Gemini）との共創設計

構造化された思考は、AIへの「プロンプト設計」において決定的な差を生みます。

LOW PURITY INPUT
「インフレについて教えて」
  • 文脈が欠如している
  • 一般的な回答しか得られない
  • 推論フローが含まれていない
RECOMMENDED
HIGH PURITY INPUT (ROOT+STEM)
「日本の家計貯蓄率が高いことが、財政赤字とインフレの関係にどう影響しているか、経済学的に説明して」
  • ROOT/STEMが定義されている
  • 推論の順序を指定する
  • 高度な洞察を生成する燃料となる
AIのアウトプットを再び「知識の木」に戻し、資産化する

思考のOSをアップデートせよ

ROOT/STEM/Branchingは、単なるメモ術ではありません。
知識を目的志向で構造化し、深化させるための思考OSです。
